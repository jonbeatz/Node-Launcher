\# Project Overview: Vader Project Engine (VPE) v2.0



\### 🛸 Core Vision

The \*\*Vader Project Engine (VPE)\*\* is a high-performance desktop command center designed for managing, repairing, and deploying local Node.js and Next.js environments. It replaces the chaos of multiple terminal windows and manual Next.js debugging with a single, persistent command interface — optimized specifically for the \*\*Vader\*\* hardware build (AMD Ryzen 9700x / Gigabyte B650) and the \*\*My Studio Channel (MSC)\*\* development workflow.



\### 🛠️ Key Components

\- \*\*Execution Engine\*\*: Uses a bundled PM2 API to ensure dev servers survive UI restarts and provide persistent process management.

\- \*\*Vader Repair Suite\*\*: Features an AST-based "Suspense Patcher" to automatically fix common Next.js 15 errors with a safe diff-view confirmation.

\- \*\*Smart Registry\*\*: Automatically detects package managers and start scripts while managing persistent project paths in `projects.json`.

\- \*\*Sleek Dashboard\*\*: A modern, dark-themed UI featuring real-time telemetry, performance sparklines, and an integrated xterm.js terminal.



\### 🎨 Design Language (Vader Protocol)

\- \*\*Aesthetic\*\*: "Studio Dark" — tactical command center with sharp, brutalist edges (0px border radius) and glassmorphic depth.

\- \*\*Palette\*\*: Main background `#121212` and surface `#1c1c1c`, accented by "Vader Red" (`#e02b20`) for critical status and actions.

\- \*\*Typography\*\*: Clean sans-serif for navigation paired with \*\*JetBrains Mono\*\* for technical data and terminal logs.

\- \*\*Branding\*\*: Strict adherence to the "msc-" prefixing for CSS and the "Powered by the MSC Media Engine" signature.



\### 🛡️ System \& Security

\- \*\*Vader Shield\*\*: Implements full context isolation and secure preload scripts via Electron.

\- \*\*OS Tuning\*\*: Specifically configured for Windows 11 25H2 with batched I/O and Ryzen 9700x hardware offloading.
